# "Não tech" versão Backend (Python)

Aqui está uma proposta de plano de ensino estruturada em 4 fases para transformar alguém "não-tech" em um pessoa desenvolvedora backend. O foco não é apenas em código, mas em como a internet funciona e como resolver problemas logicamente.

## 🏗️ Fase 1: Fundação e Pensamento Computacional

_O objetivo aqui é entender a lógica antes de se preocupar com sistemas complexos._

- **Lógica de Programação:** Variáveis, tipos de dados (strings, listas, dicionários), operadores matemáticos e lógicos.
- **Estruturas de Controle:** Condicionais (`if/else`) e laços de repetição (`for/while`).
- **Funções e Modularização:** Como organizar o código para ser reutilizável.
- **Ambiente de Desenvolvimento:** Instalação do Python, uso do VS Code e introdução ao Terminal (linha de comando).

## 🌐 Fase 2: O Ecossistema Web e Dados

_Nesta fase, a pessoa entende que o código não vive sozinho e precisa se comunicar._

- **Protocolo HTTP:** Como funciona a conversa entre o navegador (Cliente) e o servidor. Entender verbos (GET, POST, PUT, DELETE) e Status Codes (200, 404, 500).
- **Manipulação de Dados:** Leitura e escrita de arquivos JSON e CSV.
- **Bancos de Dados Relacionais (SQL):** Introdução ao PostgreSQL ou MySQL. Aprender a criar tabelas e fazer consultas básicas (SELECT, INSERT, JOIN).

## 🐍 Fase 3: Desenvolvimento Backend com Python

_Hora de colocar a mão na massa com os frameworks que o mercado utiliza._

- **Programação Orientada a Objetos (POO):** Classes, objetos, herança e encapsulamento (essencial para escalar projetos em Python).
- **Frameworks Web:** Escolha entre FastAPI (moderno e rápido) ou Django (robusto e "com tudo incluso").
- **Criação de APIs REST:** Construção de endpoints que recebem e enviam dados.
- **Autenticação e Segurança:** Noções básicas de como proteger uma rota e lidar com senhas.

## 🚀 Fase 4: Boas Práticas e Engenharia

_O que diferencia um "codificador" de um "desenvolvedor profissional"._

- **Git e GitHub:** Controle de versão, criação de branches e Pull Requests.
- **Testes Automatizados:** Introdução ao pytest (garantir que o código funciona conforme o esperado).
- **Docker:** Noções básicas de containers para que o código rode igual em qualquer máquina.
- **Metodologias Ágeis:** Como funciona o dia a dia de um time (Scrum/Kanban).
